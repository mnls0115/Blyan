🚀 Quick Setup with Docker (Recommended)

Join the network in 5 simple steps using Docker

👉 Need more details? See 📚 Common Concepts for API endpoints, security notes, and credential management

Step 1: Get Your JOIN_CODE
Request a one-time enrollment code
Your Join Code:
Valid for 30 minutes. Use it in Step 4 below.

See Common Concepts below for details on JOIN_CODE and credentials.

Step 2: Pull the Docker Image
Download the official Blyan node image
docker pull mnls0115/blyan-node:latest
Step 3: Prepare Your Environment
Set up persistent storage and networking with security hardening
# Create data directory for credentials
sudo mkdir -p /var/lib/blyan/data
sudo chmod 755 /var/lib/blyan
sudo chown $USER:$USER /var/lib/blyan/data

# Configure firewall (UFW)
sudo ufw allow 8001/tcp comment 'Blyan Node'
sudo ufw limit ssh comment 'Rate limit SSH'
sudo ufw --force enable

# Optional: Set up fail2ban for additional protection
sudo apt-get install -y fail2ban

💡 Security Tip: Consider using a dedicated user account for running the node instead of root.

Step 4: Run Your Node
Start the node with your JOIN_CODE from Step 1
docker run -d --name blyan-node \
  --gpus all \
  -p 8001:8001 \
  -v /var/lib/blyan/data:/data \
  -e JOIN_CODE=YOUR_CODE_HERE \
  -e PUBLIC_IP=$(curl -s https://checkip.amazonaws.com) \
  --restart unless-stopped \
  mnls0115/blyan-node:latest

💡 GPU Note: Omit --gpus all if you don't have NVIDIA GPU/drivers.

Step 5: Verify Enrollment & Security Status
Confirm successful network registration and security configuration
# Check enrollment status
docker logs blyan-node | grep -E "enrollment|credentials"

# Verify credential permissions
docker exec blyan-node ls -la /data/credentials.json
# Should show: -rw------- (600)

# Test health endpoint
curl http://localhost:8001/

# Check security headers
curl -I http://localhost:8001/
✓ JOIN_CODE validated successfully
✓ Node credentials obtained (node_id: gpu-xxxx)
✓ Credentials saved with correct permissions (600)
✓ Security headers configured
✓ Ready for inference requests

⚠️ Security Check: If credentials show wrong permissions, run: docker exec blyan-node chmod 600 /data/credentials.json

Step 6: Restarting Your Node
How to stop, start, and restart your node

✨ No JOIN_CODE needed for restarts!

After the first enrollment, your node has permanent credentials stored in /data/credentials.json. These are automatically loaded on every restart.

# Simple restart
docker restart blyan-node

# Or manual stop/start
docker stop blyan-node
docker start blyan-node

# View logs after restart
docker logs -f blyan-node

⚠️ When you need a new JOIN_CODE:

• If you delete /var/lib/blyan/data directory

• If you move to a new machine without copying credentials

• If credentials expire (after 90 days)

✅ That's it! Your node is now part of the Blyan network.

Essential information for all installation methods

🔑 JOIN_CODE → Permanent Credentials Flow

How nodes authenticate with the network

1. Request JOIN_CODE: Get a one-time code from this page (valid for 30 minutes)

📂 Credential Storage & Management
▼
Where your node keeps its authentication
# Docker: Mounted volume
/var/lib/blyan/data → /data/credentials.json

# Binary/Source: Local directory
./data/credentials.json

# Contents (auto-generated):

  "node_id": "gpu-xxxx-yyyy",

⚠️ Critical Security Requirements:

• File Permissions: Must be chmod 600 (owner read/write only)

• Credential Renewal: Expires after 90 days - monitor expiry date

• Backup Security: Encrypt backups and store offline

• Never: Commit to git, share publicly, or transmit unencrypted

💡 Tip: Set up monitoring to alert you 7 days before credential expiry.

🔒 Security Best Practices
▼
Essential security measures for production deployment

🔐 Authentication & Credentials

• JOIN_CODE: Single-use, 30-minute expiry, rate-limited generation

• Node Credentials: 90-day validity with JWT-based authentication

• File Permissions: Always chmod 600 /data/credentials.json

• Key Rotation: Monitor expiry and renew before expiration

• Secret Management: Use environment variables, never hardcode

🌐 Network & API Security

• TLS/SSL: All API communications use HTTPS encryption

• Firewall Rules: Only open port 8001 for node service

• IP Whitelisting: Consider restricting main server access

• DDoS Protection: Implement rate limiting at network level

• VPN Option: Use VPN for additional network isolation

🐳 Container & Runtime Security

• Non-Root User: Container runs as non-privileged user

• Read-Only Filesystem: Only /data is writable

• Resource Limits: Set CPU/memory limits to prevent abuse

• Security Scanning: Regular vulnerability scans on images

• Update Policy: Pull latest images weekly for patches

📊 Logging & Monitoring

• Log Security: Logs never contain keys or sensitive data

• Audit Trail: All enrollment/authentication attempts logged

• Health Monitoring: Regular health checks on /health endpoint

• Anomaly Detection: Monitor for unusual request patterns

• Log Retention: Keep logs for 30 days for security analysis

CRITICAL 🚨 Incident Response Plan

• Compromised Key: Immediately request new JOIN_CODE and re-enroll

• Suspicious Activity: Contact mnls0115@naver.com

• Node Quarantine: Automatic isolation of suspicious nodes

• Recovery Process: Documented steps for credential rotation

• Security Updates: Subscribe to security announcements

Alternative installation methods for binary and source deployments

📦 Binary/Source Requirements

For non-Docker deployments, you need a direct API key:

• Email mnls0115@naver.com with your GPU specs
• Receive your API key (valid for 90 days)
• Configure as environment variable: export BLYAN_API_KEY=your_key

See Common Concepts above for API endpoints and security notes.

Option A: Signed Binaries
Pre-compiled binaries with GPG signatures
# 1. Get your API key (see above)
export BLYAN_API_KEY=your_api_key_here
export MAIN_SERVER_URL=https://blyan.com/api

# 2. Download and verify binary
wget https://releases.blyan.com/latest/blyan-node-linux-amd64
wget https://releases.blyan.com/latest/blyan-node-linux-amd64.sig
gpg --verify blyan-node-linux-amd64.sig blyan-node-linux-amd64

# 3. Run the node
chmod +x blyan-node-linux-amd64
./blyan-node-linux-amd64

🔒 Always verify GPG signatures before running binaries

Option B: From Source
Clone repository and run with Python
# 1. Clone the repository
git clone https://github.com/mnls0115/Blyan.git
cd Blyan

# 2. Set up environment
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 3. Configure credentials
export BLYAN_API_KEY=your_api_key_here
export MAIN_SERVER_URL=https://blyan.com/api
export PUBLIC_IP=$(curl -s https://checkip.amazonaws.com)

# 4. Run the node
python run_blyan_node.py
